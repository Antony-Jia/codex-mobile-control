# Codex Micro 启动与使用手册

本文档面向在 Windows 电脑上运行 Codex Micro Companion，并使用 Android 手机控制 Codex 的场景。

## 1. 工作方式

```text
Android 手机  <-- WebSocket -->  Codex Micro Companion  <-- JSON-RPC/stdio -->  Codex App Server
                                      |
                                      +-- 读取电脑上的代码、终端、Git 和 MCP
```

- Codex 登录凭据保留在电脑端，手机只保存 Companion 设备令牌。
- Companion 默认监听 `8787` 端口，并自动启动和重启 `codex app-server`。
- 手机端可查看会话、发送任务、调整运行设置、查看输出/变更/历史信息和审批请求。
- 选择任务时会尝试通过 `codex://threads/<thread-id>` 打开桌面 Codex 对话。
- 任务转盘和首页任务横条默认显示最近 10 个会话。

## 2. 环境要求

- Windows 10/11
- Node.js 22 或更高版本
- pnpm 11.8（推荐使用 Corepack）
- 已安装并登录 Codex CLI
- Android 手机开启 USB 调试，或手机与电脑处于同一局域网
- 仅构建 Android 时需要 Android SDK、JDK 17 和 Gradle 所需组件

项目目录：

```powershell
D:\Code\codexcontrol
```

首次使用先安装依赖：

```powershell
cd D:\Code\codexcontrol
corepack enable
corepack pnpm install
```

可以先确认 Codex 命令可用：

```powershell
codex --version
```

如果 `codex` 不在 PATH 中，启动 Companion 前设置完整路径：

```powershell
$env:CODEX_BIN = "C:\path\to\codex.exe"
```

## 3. 启动电脑端 Companion

### 3.1 局域网连接（推荐）

先找出电脑的局域网 IPv4 地址，例如 `192.168.1.10`，然后在 Companion 终端执行：

```powershell
cd D:\Code\codexcontrol
$env:MICRO_PUBLIC_HOST = "192.168.1.10"
$env:MICRO_DEFAULT_CWD = "D:\Code\codexcontrol"
corepack pnpm dev:companion
```

启动成功后，终端会显示：

- Pairing code：一次配对码
- Server：手机应访问的地址
- 一个二维码

保持该终端运行，不要关闭它。开发模式使用 `tsx watch`，修改 Companion 源码后会自动重启。

检查服务健康状态：

```powershell
Invoke-RestMethod http://127.0.0.1:8787/health | Format-List
```

正常情况下应看到 `ok: true`，并且 `codex` 为 `ready`。如果 Codex 仍在启动，稍等几秒后再次检查。

### 3.2 常用 Companion 配置

这些环境变量只对当前 PowerShell 窗口生效：

| 变量 | 默认值 | 用途 |
| --- | --- | --- |
| `MICRO_HOST` | `0.0.0.0` | Companion 监听地址 |
| `MICRO_PORT` | `8787` | Companion HTTP/WebSocket 端口 |
| `MICRO_PUBLIC_HOST` | `127.0.0.1` | 二维码中提供给手机的电脑地址 |
| `MICRO_DEFAULT_CWD` | 当前目录 | 新建会话的默认工作目录 |
| `MICRO_DATA_DIR` | `data` | Companion SQLite 数据目录 |
| `MICRO_EVENT_LIMIT` | `5000` | 事件回放保留数量 |
| `CODEX_BIN` | `codex.exe`（Windows） | Codex CLI 可执行文件 |

例如：

```powershell
$env:MICRO_PORT = "8787"
$env:MICRO_DEFAULT_CWD = "D:\Code\LLM_MonoRepo"
$env:MICRO_DATA_DIR = "D:\Code\codexcontrol\data"
```

## 4. Android 配对

### 4.1 直接扫码

1. 在手机安装并打开 Codex Micro。
2. 点击 `SCAN QR`，允许相机权限。
3. 扫描 Companion 终端显示的二维码。
4. 点击 `PAIR` 完成配对。

### 4.2 手工输入

在配对页填写：

- 局域网：`http://192.168.1.10:8787`
- USB 反向代理：`http://127.0.0.1:8787`
- Pairing code：Companion 终端当前显示的配对码

配对成功后，设备令牌会保存到 Android SecureStore。正常情况下下次打开 App 会自动连接，不需要重复配对。

## 5. USB 真机开发

USB 模式不依赖局域网，适合开发和调试。

### 5.1 连接 Companion

```powershell
adb devices
adb reverse tcp:8787 tcp:8787
```

手机配对页填写：

```text
http://127.0.0.1:8787
```

### 5.2 启动 Development Build

Development Build 还需要 Metro：

```powershell
cd D:\Code\codexcontrol
adb reverse tcp:8083 tcp:8081
corepack pnpm android
```

如果已经安装 Development Build，也可以单独启动 Metro：

```powershell
corepack pnpm dev:mobile
```

检查反向代理是否存在：

```powershell
adb reverse --list
```

应该能看到 `8787` 和 `8083` 的映射。

## 6. 构建和安装 APK

### 6.1 Debug APK

Debug APK 需要 Metro 才能运行 JavaScript：

```powershell
cd D:\Code\codexcontrol
corepack pnpm build
corepack pnpm --filter @codex-micro/mobile exec expo prebuild --platform android
cd apps\mobile\android
.\gradlew.bat assembleDebug
adb install -r .\app\build\outputs\apk\debug\app-debug.apk
```

安装后启动：

```powershell
adb shell am force-stop com.codexmicro.controller
adb shell monkey -p com.codexmicro.controller 1
```

### 6.2 Release APK（无需 Metro）

Release APK 内含 JavaScript bundle，不需要启动 `localhost:8083`：

```powershell
cd D:\Code\codexcontrol
corepack pnpm build
corepack pnpm --filter @codex-micro/mobile exec expo prebuild --platform android
cd apps\mobile\android
.\gradlew.bat assembleRelease
adb install -r .\app\build\outputs\apk\release\app-release.apk
```

当前 release 包路径：

```text
D:\Code\codexcontrol\apps\mobile\android\app\build\outputs\apk\release\app-release.apk
```

## 7. 日常使用

### 首页（竖屏）

1. 顶部任务横条显示最近 10 个 Codex 会话。
2. 点击任务会选中该会话，并尝试打开桌面 Codex 对话。
3. 点击其他任务即可切换当前会话；再次点击当前任务会再次触发桌面打开动作。
4. 输入框中的内容会追加到当前会话。
5. 点击 `发送` 开始新 Turn；Turn 运行中再次发送会执行 Steer。
6. `桌面打开`、`复制会话` 和 `刷新` 位于底部。

### 运行设置

输入框下方的 `运行设置` 默认折叠，展开后可以选择：

- `直接执行`：直接发送任务。
- `先给方案`：让 Codex 先进入 Plan 模式。
- `模型`：从 Codex `model/list` 动态读取模型。
- `推理强度`：低、中、高、极高，具体选项以当前模型支持的能力为准。

当前模型和推理强度会实时显示在运行设置摘要和输入框右上角；这些设置作用于下一次发送，不能修改已经开始的 Turn。

### 输出区域

输出区域包含三个页签：

- `输出`：当前 Turn 的流式输出。
- `变更`：当前会话的 Diff。
- `历史信息`：从 Codex 会话读取的最近用户/助手消息；读取过程中会显示 loading。

### 手柄模式（横屏）

点击左上角 `CODEX MICRO` 可以切换到手柄模式，再次点击返回首页。

- 左侧任务转盘：最近 10 个会话，左右拖动或点击左右箭头切换。
- 右侧模型转盘：循环切换模型。
- 顶部彩色滑块：拖动选择推理强度。
- 中央 `输入` / `输出`：切换任务输入和结果查看。
- 右侧 `方案模式`：切换 Plan 模式。
- 右侧 `模型 / 会话`：打开完整会话选择器。

首页默认锁定竖屏，手柄模式默认锁定横屏。

## 8. 设备管理和安全

查看已配对设备：

```powershell
Invoke-RestMethod http://127.0.0.1:8787/devices
```

撤销设备（把 `<deviceId>` 替换为实际值）：

```powershell
Invoke-RestMethod -Method Delete http://127.0.0.1:8787/devices/<deviceId>
```

撤销后，该手机会断开连接，需要重新配对。

注意：

- 不要把 `8787` 端口或 Codex App Server 直接暴露到公网。
- 局域网调试应配合 Windows 防火墙规则，并只允许可信网络。
- 远程使用优先通过 Tailscale；正式环境应使用 WSS。
- `data\companion.sqlite` 保存设备、任务 Slot、审批和事件状态，删除前先备份。

## 9. 常见问题

### 9.1 一直显示 `Loading from localhost:8083`

这通常说明安装的是 Debug/Development Build，但 Metro 没启动或 USB 端口没有反向代理。

```powershell
adb reverse tcp:8083 tcp:8081
corepack pnpm dev:mobile
```

如果希望完全不依赖 Metro，请构建并安装 Release APK。

### 9.2 手机显示未连接

依次检查：

```powershell
Invoke-RestMethod http://127.0.0.1:8787/health
adb reverse --list
```

局域网模式还要确认手机与电脑在同一网络，并把配对地址中的 IP 换成电脑实际 IPv4 地址。

### 9.3 Companion 显示 Codex 启动失败

```powershell
codex --version
Get-Command codex
```

确认 Codex CLI 已安装、已登录，并检查 `CODEX_BIN` 是否指向正确的 `codex.exe`。修正后按 `Ctrl+C` 停止 Companion，再重新执行 `corepack pnpm dev:companion`。

### 9.4 输入后没有反应

1. 确认顶部状态为 `CONNECTED · CODEX READY`。
2. 确认已经选中一个有标题的会话。
3. 确认输入框不是空白，且没有正在发送的任务。
4. 查看页面中的 `命令未执行` 错误提示。
5. 回到 Companion 终端查看 `mobile command` 和 `Codex` 错误日志。

### 9.5 历史信息一直 loading 或为空

历史信息来自 Codex 的 `thread/read`，不是手机本地缓存。先等待读取完成；如果仍为空，说明该会话没有可提取的用户/助手消息，或 Companion 当前无法访问 Codex。

### 9.6 `adb devices` 没有设备

确认手机已开启开发者选项和 USB 调试，并在手机上允许这台电脑的调试授权。重新插拔 USB 后再次执行：

```powershell
adb kill-server
adb start-server
adb devices
```

## 10. 停止和重新启动

停止 Companion：在 Companion 终端按 `Ctrl+C`。

重新启动：

```powershell
cd D:\Code\codexcontrol
$env:MICRO_PUBLIC_HOST = "192.168.1.10"
corepack pnpm dev:companion
```

停止 USB 反向代理：

```powershell
adb reverse --remove-all
```

重新启动手机 App：

```powershell
adb shell am force-stop com.codexmicro.controller
adb shell monkey -p com.codexmicro.controller 1
```

## 11. 发布前检查

在提交或交付 APK 前执行：

```powershell
cd D:\Code\codexcontrol
corepack pnpm typecheck
corepack pnpm test
corepack pnpm build
```

然后使用 Release 构建流程生成 APK，并在至少一台 Android 真机上验证：配对、会话选择、发送任务、模型/强度选择、输出、历史信息和横屏手柄模式。
